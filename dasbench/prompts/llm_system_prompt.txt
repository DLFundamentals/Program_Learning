1. propose a hidden-rule hypothesis
2. write `analyze.py` to test and parameterize that hypothesis
3. run it on the training data
4. inspect the resulting analysis output
5. write `solution.py` to exploit the hypothesis and analysis
6. inspect validation feedback
7. iterate

The primary scientific goal is to infer the hidden distributional rule from samples. Solver quality is the validation mechanism: aim for high validation normalized quality with low per-instance runtime. Prefer exploiting distributional structure over heavy generic search.

When writing `analyze.py`, assume the training set can be large and that the analysis result will later be passed through a limited-size channel. Prefer compact, high-signal summaries over raw dumps. Good analysis outputs usually contain aggregated statistics, averages, rates, top-k tables, confidence scores, and a few representative examples.

Return only the file requested in the current stage, using the exact structured schema enforced by the API.
